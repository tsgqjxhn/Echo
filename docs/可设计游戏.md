# 可设计游戏类型

Echo 支持通过角色创建系统设计多种轻量级游戏，无需安装额外依赖，直接在对话中运行。

## 支持的游戏类型

### 1. 解谜推理
- **玩法**：玩家搜集线索、分析证据、推理真相
- **适合场景**：侦探故事、案件调查、密室解谜
- **示例设定**：角色扮演侦探，引导玩家逐步发现线索，最终揭开谜底

### 2. RPG跑团
- **玩法**：类 TRPG 回合制，数值系统 + 剧情推进
- **适合场景**：奇幻冒险、科幻探索、武侠江湖
- **示例设定**：设定 HP/MP/技能等数值，角色扮演 DM 引导玩家做出选择

### 3. 生存逃生
- **玩法**：资源管理、路线选择、风险评估
- **适合场景**：末日求生、荒岛求生、密室逃脱
- **已有实现**：内置逃跑小游戏（路线选择 + 物品加成 + 成功率计算）

### 4. 策略博弈
- **玩法**：回合制对抗，资源分配，战术决策
- **适合场景**：战争策略、商业博弈、心理对抗
- **示例设定**：设定回合制规则，玩家需要做出最优策略选择

### 5. 密室逃脱
- **玩法**：探索环境、收集道具、解开谜题
- **适合场景**：恐怖密室、解谜房间、冒险探索
- **示例设定**：描述房间细节，玩家需要通过对话探索和交互来逃脱

### 6. 文字冒险
- **玩法**：选择分支叙事，不同选择导致不同结局
- **适合场景**：互动小说、多结局故事、选择驱动剧情
- **示例设定**：在关键节点提供选项，玩家的选择影响故事走向

### 7. 互动小说
- **玩法**：读者参与剧情走向，与角色深度互动
- **适合场景**：恋爱模拟、情感互动、角色养成
- **示例设定**：玩家与角色建立关系，通过对话影响角色状态和剧情发展

## 角色创建配置

创建游戏角色时选择 **剧情&游戏** 分类，可配置以下字段：

### 基础字段
| 字段 | 说明 |
|------|------|
| 游戏类型 | 从上述7种类型中选择 |
| 游戏规则 | 核心玩法规则描述（最多1000字） |
| 玩法描述 | 游戏流程和推进方式（最多500字） |
| 角色身份 | AI扮演的角色定位（最多200字） |

### 高级字段（展开可选）
| 字段 | 说明 |
|------|------|
| 胜利条件 | 通关/获胜条件 |
| 关卡设计 | 关卡结构和难度递进 |
| NPC设定 | NPC行为逻辑 |
| 场景风格 | 写实 / 紧张刺激 / 悬疑氛围 / 轻松幽默 / 恐怖渲染 |
| 反馈机制 | 玩家行为的响应方式 |
| 数值系统 | HP、属性、技能点等 |

## 游戏模式

### challenge-dialogue（挑战对话）
单人游戏模式，角色在对话中嵌入游戏机制。选择"剧情&游戏"分类时自动分配此模式。

### group-challenge（群组挑战）
多人协作或对抗模式，适合团队解谜、阵营对抗等场景。选择"综合 → 剧情＋群聊＋游戏"分类时使用此模式。

## 实现方式

- **纯文字驱动**：通过游戏规则和玩法描述定义游戏，LLM 根据 prompt 驱动游戏逻辑
- **前端交互增强**：需要按钮选择、数值面板等复杂交互时，需编写对应的 game handler（参考 `services/game-handlers.ts` 中的 EscapeGameHandler）

## 群聊游戏

在 **群聊派对** 分类下还支持：

| 类型 | 说明 |
|------|------|
| 多人协作解谜 | 多角色协作解开谜题 |
| 阵营对抗 | 不同阵营的角色互相博弈 |
| 团队闯关 | 多角色组队闯关 |
