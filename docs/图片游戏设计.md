# 图片游戏设计

## 一、整体视觉风格指南

### 1.1 核心美学定位

**风格关键词**：现实感 × 赛博朋克 × 压抑与希望并存

参考《异次元通讯》成功经验，本作图片不追求精美立绘，而是强调**真实感与沉浸感** [2]。所有视觉元素应让玩家感觉"这是真实发生在近未来城市里的事"，而非纯虚构幻想。

### 1.2 图片风格统一规范

| 维度 | 规范要求 | 参考依据 |
| --- | --- | --- |
| 画质 | 略带噪点/压缩感，模拟手机拍摄 | 增强真实感，避免过度精致 [2] |
| 色调 | 冷色调为主（蓝灰/青绿），暖色点缀（霓虹橙/红） | 赛博朋克经典配色，呈现压抑与希望 |
| 构图 | 倾斜角度、局部特写、遮挡式构图 | 模拟紧急状态下拍摄的仓促感 |
| 光线 | 低照度环境，强对比光源（路灯/屏幕/霓虹） | 营造紧张氛围与视觉引导 |
| 元数据 | 每张图片保留时间戳、地点水印、设备信息 | 增强证据感与真实感 |
| 尺寸比例 | 9:16竖屏为主，少量1:1正方形（动态页） | 适配手机聊天界面 |

### 1.3 图片类型分类

| 类型 | 用途 | 数量占比 | 风格要求 |
| --- | --- | --- | --- |
| 场景氛围图 | 建立空间感与情绪基调 | 30% | 广角/中景，强调环境细节 |
| 线索证据图 | 提供解谜信息与剧情推进 | 45% | 特写/局部，文字清晰可读 |
| 界面截图 | 模拟系统/APP/设备界面 | 15% | UI设计需符合世界观 |
| 动态页配图 | 情感表达与角色塑造 | 10% | 可模糊/艺术化，强调情绪 |

## 三、H5小游戏设计详案（共17个）

### 3.1 游戏设计总原则

参考《异次元通讯》成功经验，本作小游戏需满足以下要求 [2]：

| 原则 | 具体要求 | 设计理由 |
| --- | --- | --- |
| **轻量嵌入** | 每个游戏时长3~15分钟，无需下载额外内容 | 保持聊天主体验流畅，避免打断沉浸感 [2] |
| **线索整合** | 游戏内容与剧情线索直接相关，成功/失败影响数值 | 避免"为游戏而游戏"，每个谜题都推进剧情 |
| **难度梯度** | 前期简单（教学），中期中等（挑战），后期困难（考验） | 配合剧情紧张感递进，避免前期劝退或后期无聊 |
| **失败惩罚** | 失败不直接GAME OVER，而是数值惩罚+剧情分支 | 降低挫败感，提高重玩价值，符合多结局设计 [2] |
| **视觉统一** | 游戏UI与聊天界面风格一致，不跳戏 | 保持沉浸感，避免"游戏里玩游戏"的割裂感 |
| **操作简化** | 以点击、拖拽、输入为主，避免复杂手势 | 适配手机操作，降低学习成本 |

## 四、动态页设计规范

### 4.1 动态页功能定位

参考《异次元通讯》动态页面设计，本作动态页承担以下功能 [2]：

| 功能 | 说明 | 设计要点 |
| --- | --- | --- |
| **情感表达** | 星发布心情、日常、想法 | 内容短小，情绪真实，不服务剧情 |
| **线索补充** | 偶尔发布含线索的照片/文字 | 线索隐蔽，不强制查看 |
| **时间流逝** | 动态发布时间与现实同步 | 增强真实感，等待机制的一部分 [2] |
| **角色塑造** | 展现星的喜好、性格、生活细节 | 甜品、夜景、吐槽等生活化内容 |
| **互动反馈** | 根据玩家选择发布不同动态 | 增强选择影响感 |

### 4.2 动态页内容规范

| 要素 | 规范要求 |
| --- | --- |
| **文字长度** | 10~50字为主，偶尔长文（100字+）用于情感爆发 |
| **发布频率** | 每章2~4条，紧急章节减少，休整章节增加 |
| **图片搭配** | 60%带图，40%纯文字，图片风格与主线一致 |
| **互动设计** | 玩家可点赞/评论（星可能回复），但不强制 |
| **删除机制** | 部分动态会在发布后自动删除（10分钟~24小时） |
| **可见范围** | 大部分公开，少数"仅玩家可见"用于特殊剧情 |

### 4.3 九部曲动态页内容表

| 章节 | 动态内容 | 配图 | 发布时间 | 特殊效果 |
| --- | --- | --- | --- | --- |

| 第三部 | "纸不会撒谎，但人会。" | 昏暗档案室照片 | 12:20 | 15分钟后删除 |
| 第四部 | "这里好像记得我，但我不记得这里。" | 透过地下通风口拍到的校园夜景 | 16:00 | 2分钟后删除 |

## 五、游戏评价标准与质量要求

### 5.1 图片质量评价标准

| 维度 | 权重 | 评分标准（1-5分） |
| --- | --- | --- |
| **真实感** | 25% | 5=完全像手机拍摄，1=过度精致像CG |
| **线索清晰度** | 20% | 5=关键信息清晰可读，1=模糊无法辨认 |
| **情绪传达** | 20% | 5=一眼感受到氛围，1=无情绪感染力 |
| **技术执行** | 15% | 5=分辨率/色彩/构图完美，1=有明显缺陷 |
| **剧情关联** | 20% | 5=与剧情紧密相关，1=可有可无 |

**及格线**：总分≥18分（满分25分）

### 5.2 小游戏质量评价标准

| 维度 | 权重 | 评分标准（1-5分） |
| --- | --- | --- |
| **玩法趣味性** | 20% | 5=有趣想重玩，1=枯燥只想跳过 |
| **难度平衡** | 20% | 5=适中有挑战，1=过难或过易 |
| **剧情整合度** | 25% | 5=与剧情深度融合，1=割裂感强 |
| **操作流畅度** | 15% | 5=操作顺畅无bug，1=频繁卡顿/误触 |
| **视觉统一性** | 10% | 5=与主界面风格一致，1=风格跳戏 |
| **反馈清晰度** | 10% | 5=成功/失败反馈明确，1=不知所云 |

**及格线**：总分≥18分（满分25分）

### 5.3 整体沉浸感评价标准

参考《异次元通讯》成功要素，本作需在以下维度达到高标准 [2]：

| 维度 | 目标 | 检测方法 |
| --- | --- | --- |
| **聊天真实感** | 玩家忘记是游戏，感觉在和真人聊天 | 用户测试反馈 |
| **时间同步感** | 玩家接受等待，感觉星有自己的生活 | 留存率/等待完成率 |
| **选择重量感** | 玩家认真考虑每个选项，不随意点击 | 选项分布统计 |
| **情感联结度** | 玩家关心星的命运，不只是想通关 | 结局选择分布/用户反馈 |
| **线索整合度** | 玩家主动整理线索，参与推理 | 论坛讨论/攻略质量 |

| 模块 | 推荐技术 | 理由 |
| --- | --- | --- |
| **游戏引擎** | Phaser.js / Pixi.js | 轻量、适配手机、性能好 |
| **UI框架** | Vue.js / React | 与聊天界面技术栈统一 |
| **数据存储** | LocalStorage + 云端备份 | 保证进度不丢失 |
| **动画效果** | CSS3 / GSAP | 流畅、性能好、易实现 |
| **适配方案** | 响应式布局 + 安全区适配 | 适配不同手机屏幕 |

### 6.3 性能优化建议

| 优化点 | 具体措施 |
| --- | --- |
| **加载速度** | 图片懒加载、游戏资源预加载、首屏≤3秒 |
| **内存占用** | 游戏结束后释放资源、图片压缩、限制同时加载数量 |
| **电量消耗** | 限制动画帧率（30fps足够）、减少后台运算 |
| **网络依赖** | 核心资源本地缓存、弱网模式降级方案 |
